# EasyTrail 合规说明

> 本文档记录项目涉及的所有合规边界与处理方式
> 最后更新:2026-07-06

## 一、法律定位

**EasyTrail 是一个临床试验信息检索工具,不是医疗服务。**

- 不提供医疗建议
- 不进行疾病诊断
- 不进行患者匹配
- 不提供诊疗指导
- 不收集用户健康信息

## 二、合规框架

### 适用法规

| 法规 | 适用范围 | 我们的应对 |
|---|---|---|
| 《互联网信息服务管理办法》 | 国内网站备案 | 域名备案(7-20 天) |
| 《广告法》医疗条款 | 任何形式的医疗宣传 | 禁用"推荐/适合/建议"等措辞 |
| 《互联网诊疗管理办法》 | 互联网诊疗活动 | 不提供任何诊疗功能 |
| 《个人信息保护法》(个保法) | 收集个人信息 | 不收集任何用户信息 |
| 《数据安全法》 | 数据处理活动 | 仅用 ClinicalTrials.gov 公开数据 |
| GDPR | 欧盟用户(若服务海外) | 不收集任何个人信息,自动合规 |
| HIPAA | 美国医疗数据 | 不接触任何 PHI,自动合规 |

## 三、核心合规机制

### 1. 顶部合规 banner(强制显示)

所有页面顶部固定显示:

```
⚠️ 本工具仅提供临床试验信息查询,不构成医疗建议。
参与临床试验请咨询主治医生。
```

### 2. 禁用措辞清单

**绝对不能使用的措辞**:
- "推荐您..."
- "适合您的..."
- "建议参与..."
- "匹配度..."
- "您应该..."
- "疗效最好..."
- "个性化推荐..."
- "智能匹配..."

### 3. 用户协议与隐私政策

- 用户协议:明确免责声明、数据来源、责任限制
- 隐私政策:明确不收集任何用户信息

### 4. ETL 数据过滤

**TIL 黑名单**:在数据同步阶段过滤掉与任职公司同类业务的试验数据,避免:
- 竞业禁止争议
- 忠实义务违反
- 商业秘密纠纷

## 四、知识产权

### 我们持有的 IP

- 源代码(MIT 协议开源)
- 品牌名/商标(待注册)
- 域名(待注册)
- 数据库结构设计

### 我们不主张的 IP

- 临床试验数据的所有权归 ClinicalTrials.gov 及原始申办方
- ClinicalTrials.gov 数据为美国政府公共领域数据
- 我们仅做信息聚合,不做内容主张

## 五、数据使用边界

### ✅ 可以做的

- 抓取 ClinicalTrials.gov 公开 API
- 缓存、索引、检索这些数据
- 通过 GitHub Pages / Cloudflare Pages 提供公开访问
- 在数据上添加中文界面、筛选功能、搜索优化

### ❌ 不能做的

- 提供医疗建议或诊疗意见
- 收集用户搜索历史或身份信息
- 修改原始数据并以官方名义发布
- 用于商业诊疗活动

## 六、特殊场景处理

### 患者询问"我应该参加哪个试验"

**正确处理**:不提供具体建议,引导用户咨询医生

```
❌ 错误: "根据您的描述,NCT07444203 适合您"
✅ 正确: "以下试验可能与您的情况相关,请咨询您的主治医生"
```

### 申办方要求下架数据

**正确处理**:ClinicalTrials.gov 数据为公开数据,我们无义务下架。但出于礼貌,可以考虑:
- 在 README 中说明数据来源和移除请求方式
- 提供 GitHub Issues 接收类似请求

## 七、当前合规风险评估

| 风险项 | 等级 | 应对 |
|---|---|---|
| 误提供医疗建议 | 🟢 低 | banner + 措辞审查 |
| 收集用户隐私 | 🟢 低 | 完全不收集 |
| 误用公司数据 | 🟢 低 | ETL 黑名单过滤 |
| 误导性信息 | 🟡 中 | 数据源标注 + 不修改原始信息 |
| 国内合规备案 | 🟡 中 | 域名备案 7-20 天 |
| 微信小程序合规 | 🟡 中 | 个人主体受限,功能严格控制 |
| 商标/域名抢注 | 🟢 低 | 待 Day 1 完成查重 |

## 八、合规审查清单

每次重大更新前,审查以下项:

- [ ] 是否仍坚持"只查询、不诊疗"定位
- [ ] 是否有新功能引入用户数据收集
- [ ] 文案是否使用禁用措辞
- [ ] ETL 过滤规则是否需要更新
- [ ] 用户协议是否需要更新
- [ ] 隐私政策是否需要更新

## 九、持续合规监控

- 每月审查一次合规清单
- 每次更新功能前评估合规风险
- 保留所有版本变更记录(通过 Git + CHANGELOG)
- 关键合规决策记录在此文件
